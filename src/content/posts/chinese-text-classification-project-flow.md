前面我们已经补齐了不少传统机器学习基础：数据清洗、特征工程、模型评估、交叉验证、调参和 Pipeline。接下来可以进入一个更像真实应用的方向：**中文文本分类项目**。

这类项目很常见，比如垃圾短信识别、评论情感分类、工单类型识别、新闻主题分类、用户反馈归类。它们看起来是在“处理文字”，但本质仍然是一个机器学习项目：输入、标签、特征、模型、评估和推理。

## 先别急着写模型

很多人做文本分类，一上来就问：用什么模型？朴素贝叶斯、逻辑回归、随机森林，还是深度学习？

这个问题当然重要，但不是第一步。第一步应该是确认：

1. 这个项目到底要判断什么；
2. 每条文本对应什么标签；
3. 数据从哪里来；
4. 输出结果给谁用；
5. 错分一条样本的代价是什么。

如果这些没想清楚，模型分数再高也可能没有意义。

## 文本分类项目的基本定义

文本分类可以写成一句话：

```text
给定一段文本，让模型判断它属于哪个类别。
```

比如：

| 输入文本 | 标签 |
| --- | --- |
| “您的快递已到达，请及时取件” | 正常短信 |
| “点击链接领取高额补贴” | 风险短信 |
| “这个功能经常闪退” | 问题反馈 |
| “界面很清爽，速度也快” | 正向评价 |

模型看到的是文本，学习目标是标签。训练完成后，再给它一条新文本，它应该输出一个类别，或者输出每个类别的概率。

## 一个完整项目有哪些环节

中文文本分类一般可以拆成这几步：

```text
原始文本
  -> 数据清洗
  -> 中文分词
  -> 去停用词
  -> 特征提取
  -> 模型训练
  -> 模型评估
  -> 推理预测
```

每一步都可能影响最终效果。

数据清洗负责去掉明显噪声，比如空文本、重复文本、无意义符号。

中文分词负责把连续句子切成词，因为中文不像英文那样天然有空格。

停用词处理负责去掉一些对分类帮助不大的常见词，比如“的”“了”“是”。

特征提取负责把文本变成数字，比如词袋、TF-IDF。

模型训练负责学习文本特征和标签之间的关系。

模型评估负责判断模型到底有没有学会，而不是只背住训练集。

## 数据表应该长什么样

最简单的数据表至少需要两列：

| text | label |
| --- | --- |
| 一段中文文本 | 类别 |

如果是真实项目，还可以增加一些辅助字段：

| 字段 | 用途 |
| --- | --- |
| id | 唯一样本编号 |
| text | 原始文本 |
| label | 人工标注类别 |
| source | 数据来源 |
| created_at | 采集时间 |
| remark | 标注备注 |

但训练模型时，最关键的仍然是 `text` 和 `label`。其他字段可以辅助排查问题，不一定直接喂给模型。

## 标签比模型更重要

文本分类项目里，标签设计经常比模型选择更难。

比如你想做“用户反馈分类”，标签可以设计成：

```text
功能问题 / 性能问题 / 体验建议 / 账号问题 / 其他
```

也可以设计成：

```text
需要处理 / 暂不处理
```

前者更细，但标注难度更高。后者更粗，但更容易落地。

标签设计要遵守三个原则：

1. 类别之间尽量互斥；
2. 每个类别有足够样本；
3. 标签要服务后续动作。

如果分类结果不会影响任何动作，那这个分类任务就没有实际价值。

## 先做一个基线模型

真实项目里，不要一开始就追求复杂模型。更稳妥的做法是先做一个 baseline。

例如：

```text
文本清洗 + 分词 + TF-IDF + 逻辑回归
```

或者：

```text
文本清洗 + 分词 + TF-IDF + 朴素贝叶斯
```

baseline 的作用不是一次做到最好，而是给项目建立一条可运行的最低版本。后面优化分词、停用词、特征、模型和参数，都可以和 baseline 对比。

## 评估不能只看准确率

如果两个类别样本数量差很多，准确率会骗人。

假设 95% 都是正常短信，只有 5% 是风险短信。模型全部预测“正常短信”，准确率也有 95%，但它对真正重要的风险短信完全没用。

所以文本分类常用这些指标：

| 指标 | 适合看什么 |
| --- | --- |
| Accuracy | 总体预测对了多少 |
| Precision | 预测为某类时有多少是真的 |
| Recall | 某类样本有多少被找出来 |
| F1 | Precision 和 Recall 的折中 |
| Confusion Matrix | 哪些类别最容易混淆 |

如果项目关注“少漏掉风险文本”，Recall 往往比 Accuracy 更重要。

## 上线时输出什么

模型训练完成后，最终一般不是只输出一个标签，还可能输出：

```json
{
  "text": "点击链接领取补贴",
  "label": "风险短信",
  "confidence": 0.91
}
```

这里的 `confidence` 可以理解为模型的把握程度。低置信度样本可以交给人工复核，高置信度样本可以自动处理。

这就把模型从“实验代码”变成了“可接入业务流程的工具”。

## 这一课先记住

中文文本分类不是先选模型，而是先定义项目。

你需要先回答：

1. 文本从哪里来；
2. 标签怎么定义；
3. 错分代价是什么；
4. 用什么流程把文本变成数字；
5. 用什么指标判断模型可不可靠。
